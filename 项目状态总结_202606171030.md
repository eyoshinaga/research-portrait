## 三、 改善方案与技术建议

### 1. 架构优化：双模存储与同步
- **方案**: 确立 MySQL 为“权威数据源”，Neo4j 为“分析数据源”。
- **实施**: 使用 Spring Event 或定时任务，当 MySQL 中新增成果时，自动更新 Neo4j 的节点与关系。

### 2. 核心算法：动态权重体系
- **时间衰减模型**: 引入 $e^{-\lambda t}$ 算法，在计算影响力时自动降低老旧成果的权重。
- **学科模板**: 在数据库中增加 `weight_config` 表，支持为不同学科（工/理/文）配置不同的指标权重。

### 3. 可视化升级：多维分析
- **雷达图**: 后端提供 6 大维度（规模、质量、影响力等）的聚合数据 API。
- **钻取功能**: 实现从“机构 -> 学院 -> 个人 -> 具体成果”的层层下钻查询逻辑。
